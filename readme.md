# Отчет по Облачной Лабораторной работе №2  
**Тема:** Сопоставление (AWS ↔ Azure) и создание кросс‑провайдерной сервисной модели.  
**Вариант:** **10**

---

## Цель работы
1. Научиться приводить данные биллинга разных провайдеров (AWS и Azure) к **единой сервисной модели**.  
2. Сформировать **vendor‑agnostic** (не зависящую от вендора) классификацию: *Compute / Storage / Network / Database / Analytics / Cloud Services*.  
3. Подготовить таблицу соответствия сервисов **AWS  Azure** (crosswalk), пригодную для анализа затрат и сравнения облаков.

---

## Исходные данные
Использованы предоставленные файлы (вариант 10):
- **AWS:** `Mapping Rules AWS team 10.csv`  
- **Azure:** `Azure lab team 10.csv`

---

## Алгоритм работы
1. Для AWS: определить сервис по **Product Code** и уточнить тип потребления по **Usage Type**.  
2. Для Azure: определить сервис по связке **Meter Category / Meter Name / Consumed Service** и выделить тип потребления.  
3. Для каждой строки заполнить сервисную модель
4. Построить таблицу соответствий **AWS Service  Azure Service** на уровне функционального назначения.
5. Сформировать сводку (количество правил, основные категории).

---

## Теория 
В реальной практике управление затратами (FinOps) и миграции требуют сравнения ресурсов **по смыслу**, а не по бренду.  
Поэтому используют **сервисную модель**, где расходы классифицируются по функциональным доменам (Compute/Storage/Network/Database/Analytics) и единице тарификации (час, GB‑month, запросы и т.д.).
